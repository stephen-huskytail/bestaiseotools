# Comparison Page Editorial Standards

This document is the source of truth for every `/comparisons/[slug]` page on BestAISEOTools.com and LocalSEOAutomation.com.

---

## 1. Target Reader + Intent

The reader lands here from a search like "Tool A vs Tool B" or "best [category] tool." They're in evaluation mode—seriously considering a purchase within the next 30 days. They've narrowed their options to 2-3 tools and want a clear answer: which one should I buy, and why?

Give them a verdict within 30 seconds of landing. Don't make them scroll past filler to get value. Respect their time and they'll trust your recommendation.

---

## 2. Required Sections (In Order)

Every comparison page must include these sections in this exact order:

### 2.1 Breadcrumbs
Visible navigation trail: `Home > Comparisons > [Tool A vs Tool B]`

### 2.2 H1 Title
Format: `[Tool A] vs [Tool B] (YYYY): [Angle/Question]`

Example: `Surfer SEO vs Clearscope (2026): Which Content Optimizer Wins?`

### 2.3 Excerpt/Meta Description
2-3 sentences. Must include both tool names. State what the reader will learn. No fluff.

### 2.4 Quick Comparison Table
Appears above the fold. See Section 3 for table rules.

### 2.5 Winner Callout
Prominent visual block declaring the winner. See Section 4 for callout rules.

### 2.6 Table of Contents
**Required for all pages 1500+ words.** Component renders conditionally at 3+ H2s, but editorial must structure content to hit that threshold for any article over 1500 words.

### 2.7 Criteria Deep Dives
Individual H2 sections for each comparison criterion. Must match the criteria in the Quick Comparison table. Each section ends with a per-criterion winner declaration.

### 2.8 Pricing Comparison
Dedicated section with current pricing tables for both tools. Include:
- All plan tiers
- Monthly/annual pricing differences
- What's included at each tier
- Any free trial or money-back guarantee info

### 2.9 Use Cases
Clear recommendations for who should choose which tool:
- "Choose [Tool A] if..."
- "Choose [Tool B] if..."

Minimum 3 bullet points per tool.

### 2.10 Real Results / Case Study
Actual performance data from using both tools. Include:
- Metrics tracked (rankings, traffic, time saved)
- Time period of testing
- Sample size
- Honest limitations of the test

No fabricated data. If you haven't tested both tools, say so and cite third-party results.

### 2.11 FAQ Section
See Section 5 for FAQ rules.

### 2.12 Final Verdict CTA
Restate the winner with primary affiliate CTA. Include secondary "but consider [Tool B] if..." option.

### 2.13 Author Bio
Byline with credentials establishing why this author can evaluate these tools.

### 2.14 Related Comparisons
Links to 3-5 related comparison articles on the site.

## 3. Comparison Table Rules

The Quick Comparison table is the highest-value element on the page. Get it right.

### Row Count
- **Minimum:** 6 criteria
- **Maximum:** 10 criteria

### Cell Content
Every cell must contain real, substantive text. **Never use `-`, `N/A`, or leave cells empty.**

If a tool doesn't have a feature:
- Say "Not available" or "Not included"
- Note the workaround if one exists
- Explain the impact of missing this feature

### Winner Declaration
Each row must declare a winner:
- **Winner badge:** Visual indicator on the winning cell
- **Tie:** Explicitly state "Tie" with a one-line reason
- **One-line reason:** Every winner/tie declaration needs a "why"

Example row:
| Criterion | Tool A | Tool B | Winner |
|-----------|--------|--------|--------|
| AI Writing | Built-in, 50+ templates | Basic suggestions only | Tool A — Full AI suite vs limited features |

### Criteria Selection
Choose criteria that matter for the purchase decision:
- Pricing (always include)
- Core differentiating features
- Ease of use
- Support/documentation
- Integration ecosystem

Avoid generic filler criteria like "Design" or "Overall Quality."

---

## 4. Winner Callout Rules

### Placement
**Always above the fold.** Reader should see the winner declaration without scrolling on desktop.

### Visual Weight
The winner callout must be visually prominent:
- Distinct background color (not white)
- Tool logo or icon
- Clear "Our Pick" or "Winner" label
- Larger text than body copy

### Required Elements
1. **Tool name** (prominently displayed)
2. **Verdict sentence:** One sentence explaining why this tool wins
3. **3 bullet reasons:** Specific, factual points supporting the verdict
4. **Primary affiliate CTA:** Button linking to the winning tool
5. **Fallback option:** "But choose [Tool B] if..." with 1-2 scenarios

Example structure:
```
[Winner Badge] Our Pick: Surfer SEO

Surfer SEO delivers better value with comparable results for most content creators.

- 50% lower starting price ($59 vs $129)
- Superior AI writing integration
- Built-in competitor analysis

[Try Surfer SEO Free →]

But choose Clearscope if: You need unlimited team seats or enterprise SSO.
```

## 5. FAQ Rules

### Question Count
- **Minimum:** 5 questions
- **Maximum:** 8 questions

### Schema Requirement
FAQs must render as accordions and generate `FAQPage` JSON-LD schema. The component must output valid structured data—not just styled HTML.

### Question Types (Required Mix)
Include at least one of each:
1. **Direct comparison:** "Is [Tool A] better than [Tool B]?"
2. **Pricing:** "Which is cheaper, [Tool A] or [Tool B]?"
3. **Use case:** "Should I use [Tool A] or [Tool B] for [specific task]?"
4. **Alternative:** "What's a good alternative to [Tool A] and [Tool B]?"
5. **Migration:** "Can I switch from [Tool A] to [Tool B]?"

### Answer Format
- 2-4 sentences per answer
- Direct answer in the first sentence
- Supporting detail in subsequent sentences
- No filler or hedging

## 6. Affiliate Link Policy

### Required Placements
Every comparison page must include affiliate links in these locations:

1. **Winner callout box** — Primary CTA button
2. **Quick Comparison table** — Tool names link to affiliate URLs
3. **Per-tool cards** — Each tool mention in "Explore These Tools" section
4. **End of FAQ** — "Ready to try [Winner]?" CTA
5. **End of article** — Final verdict with both tool CTAs

### Disclosure Requirement
Affiliate disclosure must appear:
- **Location:** First paragraph of body content (before any affiliate links)
- **Format:** Italicized, clear language
- **Text:** Include "affiliate links" and "commission" explicitly

Standard disclosure:
> *Disclosure: This article contains affiliate links. If you purchase through these links, we may earn a commission at no extra cost to you. This does not affect our editorial independence—we recommend tools based on our testing and research.*

### Link Format
Use descriptive CTA text, not generic "click here":
- Good: `[Try Surfer SEO Free →](affiliate-link)`
- Bad: `[Click here](affiliate-link)`

## 7. Length + Voice

### Word Count
- **Minimum:** 2,500 words
- **Target:** 3,500-5,000 words
- **Maximum:** 6,000 words (beyond this, split into multiple articles)

### Voice Guidelines
- **Second person:** "You" not "users" or "one"
- **Active voice:** "Surfer analyzes" not "Content is analyzed by Surfer"
- **Present tense:** Describe current features, not historical development
- **Confident:** State opinions directly. "Surfer wins" not "Surfer might be better"
- **Specific:** Numbers, feature names, and examples over generalizations

### Banned Phrases
Never use:
- "In today's fast-paced world..."
- "It goes without saying..."
- "Whether you're a beginner or expert..."
- "Without further ado..."
- "At the end of the day..."
- "Both tools are great options" (as a conclusion)
- "It depends on your needs" (without specifying the needs)
- "This tool is perfect for everyone"

## 8. SEO + Schema Checklist

### Required JSON-LD Schema
Every comparison page must output:

1. **Article** — Type, headline, author, datePublished, dateModified
2. **BreadcrumbList** — Full path from home to current page
3. **FAQPage** — All FAQ questions and answers
4. **Product or SoftwareApplication** — For each tool compared (name, brand, offers)

### Meta Title Pattern
`[Tool A] vs [Tool B] (YYYY): [Angle/Hook]`

- Total length: 50-60 characters
- Year: Always current year
- Angle: Question or value proposition

Examples:
- `Surfer SEO vs Clearscope (2026): Which Content Optimizer Wins?`
- `Ahrefs vs Semrush (2026): Best All-in-One SEO Tool?`

### Meta Description Pattern
- Length: 150-160 characters
- Include both tool names
- State the verdict or key differentiator
- Include a micro-CTA

Example:
> We tested Surfer SEO vs Clearscope for 3 months. Winner: Surfer for most users (better AI, lower price). See the full breakdown.

### Internal Linking
- Link to individual tool review pages
- Link to related comparison articles
- Link to relevant category/guide pages
- Minimum 5 internal links per article

## 9. Visual / UX Requirements (Editorial Responsibility)

### Featured Image
- **Required:** Yes, every comparison page
- **Dimensions:** 1200x675px minimum (16:9 aspect ratio)
- **Alt text:** Must include both tool names: "Comparison of [Tool A] vs [Tool B] interfaces"
- **Style:** Split-screen showing both tools, or side-by-side logos

### Tool Screenshots
- **Required:** At least one screenshot per tool
- **Content:** Show the primary interface the article discusses
- **Captions:** Descriptive caption explaining what the screenshot shows
- **Freshness:** Screenshots must be from the current tool version (check quarterly)

### Logo Usage
- Use official tool logos only
- Link to brand asset pages when available
- Don't modify, recolor, or distort logos
- Include proper spacing around logos

## 10. Reject List (Anti-Patterns)

These patterns are explicitly banned. Fix immediately if found.

### Table Issues
- **Empty cells:** Cells with `-`, `N/A`, or blank content
- **Missing winners:** Rows without a declared winner or tie
- **Generic criteria:** "Overall Quality" or "Design" without specifics
- **Outdated pricing:** Prices more than 3 months old

### Winner Callout Issues
- **Below the fold:** Winner not visible without scrolling
- **Text-only:** No visual distinction from body content
- **Missing CTA:** No affiliate button in the winner box
- **No reasons:** Winner declared without supporting points
- **Wishy-washy:** "Both are great, it depends" conclusions

### FAQ Issues
- **No schema:** FAQ exists but doesn't generate JSON-LD
- **No accordions:** FAQ renders as flat text, not expandable
- **Generic questions:** "What is [Tool A]?" — that's a review question, not comparison
- **Too few:** Fewer than 5 questions

### Content Issues
- **Fabricated data:** Made-up test results or statistics
- **Missing disclosure:** No affiliate disclosure before first link
- **Outdated features:** Describing deprecated or changed functionality
- **Generic conclusions:** "Both tools are excellent choices for anyone"
- **Feature lists only:** No original testing, analysis, or opinion

### CTA Issues
- **Sparse CTAs:** Only one affiliate link in the entire article
- **Generic labels:** "Click here" or "Learn more" instead of tool name
- **Missing fallback:** No "but consider X if..." alternative option

### Structure Issues
- **Missing sections:** Skipping required sections from the template
- **Wrong order:** Sections appearing out of specified sequence
- **No TOC:** Article over 1500 words without table of contents
- **Misleading labels:** "Explore These Tools" for internal links (should be clearer)

---

## Enforcement

Before publishing any comparison page:

1. Run through this checklist section by section
2. Verify all schema outputs in Google's Rich Results Test
3. Check affiliate links resolve correctly
4. Confirm featured image displays properly
5. Test TOC functionality
6. Validate FAQ accordion behavior

Pages that fail any section should not go live until fixed.
